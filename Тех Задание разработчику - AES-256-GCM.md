# ТЗ - AES-256-GCM

## 1. Назначение

Разработано настольное локальное приложение для шифрования и дешифрования текстовых данных по паролю.

Основная цель:
- безопасно шифровать текст;
- безопасно дешифровать ранее созданные данные;
- не требовать от пользователя ручного ввода `IV`, `nonce`, `salt` и `tag`.

Дополнительная цель:
- зафиксировать проект так, чтобы при утрате исходников можно было восстановить полностью совместимое приложение.

## 2. Платформа и стек

- язык: `Python 3.13`
- GUI: `tkinter`
- криптография: `cryptography`
- режим шифрования: `AES-256-GCM`
- получение ключа из пароля: `PBKDF2-HMAC-SHA256`
- кодировка текста: `UTF-8`
- формат результата: `Base64`

## 3. Состав приложения

Приложение состоит из:
- одного основного окна;
- системных окон предупреждений и ошибок;
- режима командной строки.

## 4. Заголовок окна

Точный заголовок главного окна:

## 6. Поведение элементов

### 6.1. Исходные данные

Поле используется:
- для открытого текста перед шифрованием;
- для строки `Base64` перед дешифрованием.

Поле должно поддерживать:
- многострочный текст;
- вставку мышью и клавиатурой;
- выделение, копирование, вырезание, вставку.

### 6.2. Пароль

Поле используется для ввода пароля.

Требования:
- по умолчанию пароль скрыт символом `*`;
- чекбокс `Показать пароль` показывает и скрывает пароль;
- поле поддерживает копирование, вставку, вырезание и выделение.

### 6.3. Результат

Поле используется:
- для вывода строки `Base64` после шифрования;
- для вывода расшифрованного текста после дешифрования.

Поле поддерживает:
- копирование;
- вставку;
- выделение;
- контекстное меню.

## 7. Поведение кнопок

### 7.1. Зашифровать

Кнопка должна:
- взять текст из поля `Исходные данные`;
- взять пароль из поля `Пароль`;
- сгенерировать `salt`;
- вывести ключ из пароля;
- сгенерировать `nonce`;
- зашифровать текст через `AES-256-GCM`;
- собрать бинарный пакет;
- вывести результат в `Base64` в поле `Результат`.

### 7.2. Расшифровать

Кнопка должна:
- взять строку из поля `Исходные данные`;
- взять пароль из поля `Пароль`;
- декодировать `Base64`;
- разобрать пакет;
- восстановить ключ из пароля;
- выполнить дешифрование;
- вывести открытый текст в поле `Результат`.

### 7.3. Копировать результат

Кнопка должна копировать всё содержимое поля `Результат` в буфер обмена.

### 7.4. Вставить

Кнопка должна вставлять содержимое буфера в активное поле.

Если активного поля нет, вставка должна выполняться в `Исходные данные`.

### 7.5. Поменять местами

Кнопка должна менять местами содержимое полей `Исходные данные` и `Результат`.

### 7.6. Очистить

Кнопка должна:
- очищать `Исходные данные`;
- очищать `Результат`;
- не очищать пароль.

## 8. Горячие клавиши и мышь

## 9. Криптографические требования

Основной режим приложения:
- `AES-256-GCM`

Параметры:
- `salt`: `16 байт`
- `nonce`: `12 байт`
- `PBKDF2` итерации: `200000`
- хэш для `PBKDF2`: `SHA-256`
- длина ключа основного режима: `32 байта`

Пользователь вручную не вводит:
- `IV`
- `nonce`
- `salt`
- `tag`

## 10. Совместимость

Приложение создаёт новые данные только в режиме `AES-256-GCM`.

Для обратной совместимости дешифратор понимает коды размеров ключа:
- `1` = `128 бит`
- `2` = `192 бита`
- `3` = `256 бит`

Новая запись должна использовать код `3`.

## 11. Обработка ошибок

Приложение должно показывать понятные сообщения для пользователя в случаях:
- не введён текст для шифрования;
- не введён пароль;
- строка для дешифрования не является корректным `Base64`;
- формат пакета неизвестен;
- зашифрованные данные слишком короткие;
- размер ключа в пакете неизвестен;
- пароль неверный;
- шифротекст повреждён.

## 12. Режим командной строки

Поддерживаются команды:

```powershell
py aes256_tool.py encrypt --password "пароль" --text "текст"
py aes256_tool.py decrypt --password "пароль" --data "BASE64"
```

Поддерживается дополнительный аргумент совместимости:

```powershell
--size 128|192|256
```

По умолчанию используется `256`.

## 13. Критерий готовности

Приложение считается готовым, если:
- шифрует текст по паролю;
- дешифрует свои данные без потерь;
- корректно работает с русским текстом;
- корректно работает с многострочным текстом;
- корректно работает с буфером обмена;
- показывает понятные ошибки;
- проходит автоматические тесты и ручную проверку.
