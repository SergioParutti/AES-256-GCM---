# Инструкция по восстановлению приложения

## 1. Цель

Инструкция описывает, как заново создать полностью совместимое приложение, если исходный проект будет утрачен.

Совместимость означает:
- новая реализация должна открывать и дешифровать данные, созданные исходным приложением;
- интерфейс и сценарии работы должны быть максимально близки к исходной версии.

## 2. Что нужно восстановить

Нужно восстановить:
- GUI-приложение на `Python`
- CLI-режим
- формат шифропакета
- автоматические тесты

## 3. Требования к окружению

- `Windows`
- `Python 3.13`
- установленный `pip`
- библиотека `cryptography`

Установка зависимости:

```powershell
py -m pip install -r requirements.txt
```

Или:

```powershell
py -m pip install cryptography==46.0.5
```

## 4. Минимальная структура проекта

Достаточно восстановить следующие файлы:
- `aes256_tool.py`
- `test_aes256_tool.py`
- `requirements.txt`
- `readme.md`

Для полной восстановленной версии дополнительно должны быть:
- `ТЗ - AES-256-GCM.md`
- `Спецификация формата шифропакета.md`
- `Описание проекта.md`
- `Инструкция по восстановлению приложения.md`

## 5. Что должно быть реализовано в коде

В `aes256_tool.py` обязательно должны быть:
- функция проверки размера ключа;
- функция вывода ключа из пароля через `PBKDF2-HMAC-SHA256`;
- функция шифрования текста;
- функция дешифрования текста;
- GUI на `tkinter`;
- CLI-команды `encrypt` и `decrypt`.

## 6. Обязательные параметры совместимости

Новая реализация обязана использовать:
- маркер формата `P2`
- `salt` длиной `16 байт`
- `nonce` длиной `12 байт`
- `PBKDF2` с `200000` итерациями
- `SHA-256` в составе `PBKDF2`
- основной ключ `256 бит`
- код основного размера ключа `3`
- `UTF-8` для текста
- `Base64` как внешний формат

Точный формат пакета описан в документе `Спецификация формата шифропакета.md`.

## 7. Обязательный интерфейс

Восстановленное приложение должно иметь:
- заголовок `AES-256-GCM шифрование и дешифрование by Serjio Parutti 2026`
- поле `Исходные данные`
- поле `Пароль`
- чекбокс `Показать пароль`
- поле `Результат`
- кнопки:
  - `Зашифровать`
  - `Расшифровать`
  - `Копировать результат`
  - `Вставить`
  - `Поменять местами`
  - `Очистить`

## 8. Команды запуска

GUI:

```powershell
py aes256_tool.py
```

CLI шифрование:

```powershell
py aes256_tool.py encrypt --password "пароль" --text "текст"
```

CLI дешифрование:

```powershell
py aes256_tool.py decrypt --password "пароль" --data "BASE64"
```

## 9. Команда проверки

Автотесты:

```powershell
py -m unittest test_aes256_tool.py
```

## 10. Обязательная проверка совместимости

После восстановления нужно проверить:
- успешное шифрование и дешифрование обычного текста;
- успешную работу с русским текстом;
- успешную работу с многострочным текстом;
- ошибку на неверном пароле;
- ошибку на битом `Base64`;
- совпадение с эталонным тестовым вектором из спецификации.

## 11. Критерий успешного восстановления

Приложение считается восстановленным корректно, если:
- интерфейс соответствует описанию;
- команды запуска работают;
- автотесты проходят;
- эталонный тестовый вектор совпадает;
- новый аналог дешифрует старые данные без изменений формата.
