# Спецификация формата шифропакета

## 1. Назначение

Документ фиксирует точный формат данных, создаваемых приложением, чтобы по нему можно было реализовать полностью совместимый дешифратор или аналогичное приложение.

## 2. Общая схема

Шифрование выполняется так:
- пароль пользователя переводится в ключ через `PBKDF2-HMAC-SHA256`;
- данные шифруются через `AES-GCM`;
- результат собирается в бинарный пакет;
- бинарный пакет кодируется в `Base64`.

## 3. Параметры

- алгоритм шифрования: `AES-GCM`
- основной размер ключа: `256 бит`
- кодировка открытого текста: `UTF-8`
- `salt`: `16 байт`
- `nonce`: `12 байт`
- тег аутентификации: встроен в конец шифротекста
- длина тега `GCM`: `16 байт`
- `PBKDF2` итерации: `200000`
- хэш `PBKDF2`: `SHA-256`

## 4. Бинарный формат пакета

Бинарный пакет имеет следующую структуру:

- байты `0-1`: маркер формата `P2`
- байт `2`: код размера ключа
- байты `3-18`: `salt`
- байты `19-30`: `nonce`
- байты `31-end`: `ciphertext || tag`

Обозначения кодов размера ключа:
- `1` = `128 бит`
- `2` = `192 бита`
- `3` = `256 бит`

Для новых данных должно использоваться значение `3`.

## 5. Формирование ключа

Ключ формируется из пароля по схеме:

- алгоритм: `PBKDF2-HMAC-SHA256`
- пароль: строка пользователя в `UTF-8`
- `salt`: `16 байт`
- количество итераций: `200000`
- длина ключа:
  - `16 байт` для кода `1`
  - `24 байта` для кода `2`
  - `32 байта` для кода `3`

## 6. Шифрование

При шифровании:
- открытый текст переводится в `UTF-8`;
- создаётся случайный `salt`;
- создаётся случайный `nonce`;
- вычисляется ключ;
- вызывается `AESGCM(key).encrypt(nonce, plaintext_bytes, None)`;
- в результат `encrypt` уже входит тег аутентификации;
- пакет собирается как `header + salt + nonce + ciphertext_with_tag`;
- пакет кодируется через стандартный `Base64`.

## 7. Дешифрование

При дешифровании:
- входная строка декодируется как строгий `Base64`;
- проверяется минимальная длина пакета;
- проверяется маркер `P2`;
- читается код размера ключа;
- извлекаются `salt`, `nonce`, `ciphertext_with_tag`;
- из пароля повторно строится ключ;
- вызывается `AESGCM(key).decrypt(nonce, ciphertext_with_tag, None)`;
- результат переводится из `UTF-8` в строку.

## 8. Правила валидации

Должны выполняться следующие проверки:
- строка должна быть корректным `Base64`;
- пакет должен быть не короче `31 + 16` байт;
- маркер должен быть равен `P2`;
- код размера ключа должен быть одним из `1`, `2`, `3`;
- при ошибке аутентификации дешифрование должно завершаться ошибкой.
