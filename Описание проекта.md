# Описание проекта

## 1. Что это за проект

Проект представляет собой локальное приложение для шифрования и дешифрования текста по паролю.

Приложение рассчитано на обычного пользователя:
- без ручного ввода криптографических параметров;
- без выбора сложных режимов;
- с простым окном и понятными кнопками.

## 2. Что умеет приложение

- шифровать текст в строку `Base64`
- дешифровать ранее созданную строку `Base64`
- работать с многострочным текстом
- работать с русским текстом
- копировать и вставлять данные мышью и клавиатурой
- запускаться как GUI и как CLI

## 3. Что не входит в проект

- шифрование файлов
- сетевой обмен
- хранение паролей
- ручной ввод `IV`, `nonce`, `salt`, `tag`
- выбор алгоритма пользователем

## 4. Архитектура

Проект простой и однофайловый.

Основные части:
- `aes256_tool.py` — логика шифрования, дешифрования, GUI и CLI
- `test_aes256_tool.py` — автоматические тесты
- `readme.md` — краткая инструкция
- `requirements.txt` — зависимость проекта
- `ТЗ - AES-256-GCM.md` — финальное техническое задание
- `Спецификация формата шифропакета.md` — точный формат данных
- `Инструкция по восстановлению приложения.md` — как восстановить аналог

## 5. Основной сценарий пользователя

Шифрование:
- ввести текст;
- ввести пароль;
- нажать `Зашифровать`;
- скопировать строку `Base64`.

Дешифрование:
- вставить строку `Base64`;
- ввести тот же пароль;
- нажать `Расшифровать`;
- получить исходный текст.

## 6. Техническая основа

Приложение использует:
- `Python 3.13`
- `tkinter`
- `cryptography`
- `AES-256-GCM`
- `PBKDF2-HMAC-SHA256`

## 7. Особенность проекта

Проект документируется не только для разработки, но и для восстановления.

Это означает, что документация должна позволять:
- повторно реализовать приложение;
- сохранить совместимость формата;
- дешифровать старые данные новым аналогом.
